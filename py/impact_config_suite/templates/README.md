# HTML Compare Report Template System

This directory contains separated HTML, CSS, and JavaScript files for the comparing HTML report generation system. This makes the code much easier to edit, maintain, and update.

## File Structure

```
templates/
├── report_template.html    # Main HTML structure (Jinja2 template)
├── report.css              # All styling (can be edited directly)
├── report.js               # All JavaScript (can be edited directly)
└── README.md               # This file
```

## Quick Reference

### report_template.html
- **Purpose**: Main HTML structure using Jinja2 templating
- **Variables available**:
  - `{{ summary }}`: Report summary text
  - `{{ generated }}`: Generation timestamp
  - `{{ column_headings }}`: List of column headers
  - `{{ rows_body }}`: HTML table body rows
  - `{{ delmerge_section }}`: Optional parent div preview section
- **Edit for**: Page layout, structure changes, HTML organization
- **Auto-copy**: Linked in report via `<link rel="stylesheet" href="report.css">` and `<script src="report.js"></script>`

### report.css
- **Purpose**: Complete styling for the report
- **Includes**: Table styles, filters, find-bar, delmerge preview section
- **Auto-copy**: Copied to the export folder when report is generated
- **Edit for**: Colors, fonts, spacing, responsive design
- **Note**: No need to restart Python - changes take effect on next export

### report.js
- **Purpose**: Complete JavaScript functionality
- **Features**:
  - Main report filtering (`af()`, `bd()`)
  - Find functionality (`df()`, `fp()`, `fn()`, `cf()`)
  - Delmerge navigation (`gotoDelmergeNext()`, `gotoDelmergePrev()`)
  - HTML view toggle (`toggleHtmlView()`, `toggleHtmlViewDelmerge()`)
  - Event binding
- **Auto-copy**: Copied to the export folder when report is generated
- **Edit for**: Interaction logic, UI behavior
- **Note**: No need to restart Python - changes take effect on next export

## How Python Integrates These Files

### Normal Flow (Jinja2 installed)
```
Python: _export_html_report()
  → Loads templates/ folder path
  → Creates Jinja2 environment
  → Loads report_template.html
  → Renders with context (summary, rows, etc.)
  → Copies report.css and report.js to export folder
  → Exports HTML with linked CSS/JS
```

### Fallback Flow (Jinja2 not available)
```
Python: _export_html_report()
  → Catches exception
  → Calls _generate_html_fallback()
  → Inlines minimal CSS directly in HTML
  → Exports standalone HTML (no CSS/JS files)
  → Shows warning to user
```

## Installation Requirements

### Jinja2 (Recommended)
```bash
pip install jinja2
```

If Jinja2 is not installed, the system will still work but will fallback to inlining minimal CSS (less aesthetic but functional).

## Editing Workflow

### To Change Styling
1. Edit `report.css`
2. Export report - styles are included and copied to export folder
3. No Python restart needed

### To Change Layout/Structure
1. Edit `report_template.html` - Jinja2 variables are available
2. Export report - new structure is rendered
3. No Python restart needed

### To Change Interactivity
1. Edit `report.js`
2. Export report - new script is copied to export folder
3. Open report in browser - new functionality available

## Python Code References

## Example: Adding a New CSS Class

### Current HTML (generated by Python)
```html
<tr data-status="Same" ...>
```

### Step 1: Add CSS in report.css
```css
tr[data-status="Same"].highlight {
  background: #ffd700 !important;
  font-weight: bold;
}
```

### Step 2: Update JavaScript (report.js)
```javascript
// Add to toggleDelmergeViewStyle function:
mainRows.forEach(function(r){
  if(r.dataset.status === "Same")
    r.classList.add('highlight');
});
```

### Step 3: Export Report
Python automatically:
- Renders template with new CSS  
- Copies updated reportCSS to export folder
- Report displays with new styling

## Troubleshooting

### CSS/JS Changes Not Showing
- **Issue**: Old files cached in browser
- **Solution**: Hard refresh browser (Ctrl+Shift+R or Cmd+Shift+R)
- **Or**: Check that CSS/JS files were copied to export folder

### Jinja2 Template Error
- **Issue**: "Jinja2 templating failed..."
- **Solution**: Install with `pip install jinja2`
- **Or**: Leave as-is - fallback HTML will still work

### Export Folder Missing CSS/JS
- **Issue**: Files not copied
- **Solution**: Check folder permissions
- **Verify**: `_copy_template_assets()` runs in `_export_html_report()`

## Best Practices

✅ **DO**:
- Use template variables (`{{ variable }}`) - they're auto-escaped
- Keep CSS organized by component (filters, table, delmerge, etc.)
- Use `| safe` filter only for HTML that's already escaped
- Test CSS changes locally before export
- Comment complex JavaScript functions

❌ **DON'T**:
- Hardcode data in templates - use Jinja2 variables
- Modify Python files for styling - use CSS
- Remove `| safe` filters without understanding XSS risks
- Export to production without testing in browser
